## 3. Что тестировать прямо сейчас

### Скрипт 1 — `generate_pdf.py`

**Что делает:** Генерирует PDF-отчёт «Исследование рекламных каналов» (сравнительная таблица бесплатных и платных инструментов).  
**Выходной файл:** `advertising_channels_report.pdf`

**Запуск:**

```bash
source .venv/bin/activate
python generate_pdf.py
```

**Ожидаемый вывод в консоль:**

```
============================================================
PDF Generator (Vaysberg v2)
============================================================
Creatung PDF file...
PDF created: /home/deploy/projects/vaysberg-v2/advertising_channels_report.pdf
Size: X.XX MB

Success! PDF is ready.
============================================================
```

> **Известная проблема:** Строка `Creatung PDF file...` — опечатка в коде (`generate_pdf.py`, строка 169). Должно быть `Creating`. Баг зафиксирован в разделе 5.

**Проверка результата:**

```bash
ls -lh advertising_channels_report.pdf
# Файл должен существовать и иметь размер > 0
```

Открыть PDF и убедиться:
- [ ] Файл открывается без ошибок
- [ ] Заголовок: «Исследование рекламных каналов»
- [ ] Присутствуют таблицы (таблица текущего стека, сравнительная таблица)
- [ ] Данные в таблицах читаемы

> **Известная проблема:** Весь текст в файле — латинская транслитерация кириллицы (`Kanal`, `Instrument`, `Sajt`). Это не баг отображения — так написан сам код. Шрифт `Helvetica-Bold` не поддерживает кириллицу, поэтому русский текст в данном скрипте не используется. Для сравнения: `generate_glc_reputation_pdf.py` правильно регистрирует шрифт DejaVu и использует полноценную кириллицу.

## 5. Реестр найденных дефектов

### BUG-001 — Опечатка в выводе консоли

| Поле | Значение |
|---|---|
| Файл | `generate_pdf.py` |
| Строка | 169 |
| Серьёзность | Low |
| Тип | Косметический |

**Текущий код:**
```python
print(f"Creatung PDF file...")
```

**Должно быть:**
```python
print(f"Creating PDF file...")
```

---

### BUG-002 — Отсутствие кириллической поддержки в `generate_pdf.py`

| Поле | Значение |
|---|---|
| Файл | `generate_pdf.py` |
| Строки | 35-72, 91-97 |
| Серьёзность | Medium |
| Тип | Функциональный |

**Описание:** Скрипт использует шрифты `Helvetica-Bold` и `Helvetica` (строки стилей `fontName`), которые не поддерживают кириллицу. Весь контент написан в латинской транслитерации. В отличие от него, `generate_glc_reputation_pdf.py` регистрирует шрифт DejaVu (строки 21-24) и корректно работает с русским текстом.

**Воспроизведение:** Попробуйте добавить кириллический текст в любой `Paragraph()` — в PDF появятся символы-замена или пустые места.

**Ожидаемое поведение:** Оба скрипта должны поддерживать кириллицу через шрифт DejaVu.

---

### BUG-003 — Жёстко прописанный путь к шрифтам

| Поле | Значение |
|---|---|
| Файл | `generate_glc_reputation_pdf.py` |
| Строка | 21 |
| Серьёзность | Medium |
| Тип | Портируемость |

**Текущий код:**
```python
FONTS_DIR = Path("/usr/share/fonts/truetype/dejavu")
```

**Описание:** Путь захардкожен под Linux/Debian. На macOS шрифты находятся в `/Library/Fonts/`, на Windows — в `C:\Windows\Fonts\`. Скрипт упадёт на любой не-Linux системе.

**Ожидаемое поведение:** Путь к шрифтам должен определяться динамически или быть вынесен в конфигурацию.

---

### BUG-004 — Отсутствует `requirements.txt`

| Поле | Значение |
|---|---|
| Серьёзность | High |
| Тип | Инфраструктурный |

**Описание:** В проекте нет файла `requirements.txt`. Новый разработчик не сможет воспроизвести окружение без него. Зависимости зафиксированы только в `.venv/`.

**Ожидаемое поведение:** Файл `requirements.txt` должен существовать. Можно сгенерировать:

```bash
source .venv/bin/activate
pip freeze > requirements.txt
```

---

### BUG-005 — Отсутствует `.env.example`

| Поле | Значение |
|---|---|
| Серьёзность | High (критично для следующих фаз) |
| Тип | Инфраструктурный |

**Описание:** Согласно `brief.md`, приложение потребует множество API-ключей. Нет ни `.env`, ни `.env.example`, ни шаблона переменных окружения.

**Ожидаемое поведение:** Должен существовать `.env.example` со всеми переменными (без реальных значений). Реальный `.env` должен быть в `.gitignore`.

### Фаза 1 — MVP (планируется)

После реализации появятся следующие объекты тестирования:

**API-эндпоинты (FastAPI):**

- `POST /api/research/{project_id}/analyze` — запустить анализ
- `GET /api/research/{project_id}/results` — получить результаты

- `GET /health` — health check

**Парсеры:**
- Парсер сайта (BeautifulSoup + Playwright)
- Парсер VK (VK API v5)
- Парсер Instagram (Instaloader)

**Требуемые переменные окружения для Фазы 1:**

```env
CLAUDE_API_KEY=

```

**Планируемая структура тестов:**
```
tests/

  integration/   # Тесты API-эндпоинтов

```

**Запуск тестов (когда будет реализовано):**

### Обработка ошибок — что проверять

Согласно `brief.md`, архитектура предусматривает три класса ошибок:

| Класс | Поведение | Что проверять |
|---|---|---|
| `FatalError` | Стоп всего анализа | Неверный API-ключ, нет интернета |
